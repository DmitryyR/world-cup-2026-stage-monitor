## Reviewer Output

The live app already looks like a credible dark sports operations dashboard, but the audit found a trust problem in the accepted data presentation.

Observed issues:

- Summary and Bracket showed "Next Match" as a scheduled match with a kickoff date that was already in the past.
- Upcoming Matches also included past scheduled fixtures, so the app looked like it was calmly waiting for a match that had already left the station.
- Data Health showed a passed/data-ok state even when accepted scheduled matches appeared stale.
- Matches page displayed some future dependency labels as generic "Winner of Match 93" style placeholders where the app had enough data to show a friendlier label.
- Team Path could treat stale scheduled matches as the next step.

What worked:

- Navigation, main pages, dark dashboard layout, Kyiv time labels, flag rendering, status badges, and bracket structure were functional.
- Maker/checker architecture and persisted accepted data flow remained visible.
- The UI had a strong visual direction and did not feel like a raw generated prototype.

## Tasks Selected For This Iteration

- Exclude past scheduled matches from Summary and Bracket "Next Match" / "Upcoming Matches".
- Mark scheduled matches whose kickoff time passed by more than 4 hours as "Needs review" in display logic.
- Surface stale scheduled matches in Data Health diagnostics.
- Show "Scheduled time passed" instead of a generic scheduled state where relevant.
- Resolve match dependency labels in the Matches table to friendlier names, for example "Winner of Portugal / Spain" when source match data is available.
- Keep the changes limited to UI/product diagnostics and display helpers; no provider, checker, stage detector, monitor loop, or schema changes.
